# Anti-AI-Slop Patterns

Questi pattern non sono errori automatici. Diventano issue quando riducono
chiarezza, fiducia, usabilita' o coerenza del prodotto.

## Pattern Da Cercare

- Gradienti generici e decorativi senza funzione.
- Card annidate inutilmente.
- Icone decorative ripetitive.
- Gerarchia piatta: tutto ha lo stesso peso.
- Grigi poco leggibili o contrasto insufficiente.
- CTA vaghe come "Scopri di piu'" o "Continua" senza contesto.
- Spacing casuale tra sezioni, card e controlli.
- Font o scelte tipografiche generiche non legate al dominio.
- Hero troppo template, senza prodotto, dati o stato reale.
- Dashboard piena ma senza priorita' operativa.
- Troppi badge, pill o chip non azionabili.
- Shadow, border e rounded incoerenti tra componenti simili.
- Empty state senza azione o spiegazione.
- Error state impersonale o non risolvibile.
- Loading state che cambia layout o nasconde progresso.
- Microcopy impersonale, promozionale o non specifica.
- Componenti simili ma non uguali senza motivo.
- Palette monotona che rende impossibile distinguere stati.
- Immagini decorative che non aiutano a capire prodotto o task.

## Come Valutarli

- Collegare ogni pattern a impatto utente, non a gusto personale.
- Dare priorita' alta solo se il pattern compromette task, fiducia o
  accessibilita' base.
- Preferire fix piccoli: rimuovere decorazione inutile, rafforzare gerarchia,
  chiarire CTA, uniformare componenti.
- Verificare dopo il fix con screenshot, viewport target o review statica.

## Segnali Di UI Piu' Solida

- Il task primario e' evidente.
- La pagina usa dati, stati e contenuti realistici.
- Le azioni hanno testo specifico.
- Gli stati critici sono piu' leggibili degli elementi decorativi.
- Componenti simili condividono struttura e comportamento.
- La UI regge senza effetti visuali superflui.

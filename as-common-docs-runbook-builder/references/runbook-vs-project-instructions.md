# Runbook Vs Project Instructions

## Usa runbook builder quando

- Serve una procedura ripetibile.
- Serve una guida di manutenzione.
- Serve troubleshooting.
- Serve onboarding operativo.
- Serve documentare deploy, verifica, rollback o routine.

## Usa project instructions builder quando

- Alberto chiede Project Instructions ChatGPT.
- Si deve scrivere o rivedere AGENTS.md come policy agente.
- Si definiscono regole Codex, quality gate e comportamento degli assistenti.
- Si separano istruzioni globali, repository e Copilot.

## Usa riepilogo operativo quando

- Alberto vuole chiudere una chat lunga.
- Serve un prompt di ripartenza per una nuova chat.
- Il contenuto non deve diventare documentazione persistente di repository.

## Regola pratica

Runbook = come eseguire una procedura.
Project instructions = come deve comportarsi l'assistente nel progetto.
Riepilogo operativo = come ripartire da una chat.
